# Contributing to Internara

Thank you for considering contributing to Internara! We welcome contributions from everyone.

## Getting Started

1.  **Fork the repository** on GitHub.
2.  **Clone your fork** locally.
3.  **Create a new branch** for your feature or bug fix (`git checkout -b feature/amazing-feature`).
4.  **Make your changes**. Please adhere to the coding conventions outlined in
    `docs/main/development-conventions.md`.
5.  **Run tests** to ensure your changes don't break existing functionality (`php artisan test`).
6.  **Commit your changes** with descriptive commit messages.
7.  **Push to your fork** (`git push origin feature/amazing-feature`).
8.  **Submit a Pull Request** to the main repository.

## Coding Standards

- Follow the **PSR-12** coding standard.
- Use **Pest** for testing.
- Adhere to the **Modular Monolith** architecture described in `docs/main/architecture-guide.md`.
- Ensure your code is formatted using **Laravel Pint** (`vendor/bin/pint`).

## Reporting Bugs

If you find a bug, please open an issue on GitHub with a detailed description, steps to reproduce,
and any relevant logs or screenshots.

## Security Vulnerabilities

If you discover a security vulnerability, please send an email to reasnov.official@gmail.com instead
of using the issue tracker. All security vulnerabilities will be promptly addressed.
