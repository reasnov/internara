# Support & Communication: Internara Project

This file provides information on how to get help, report issues, and communicate with the Internara community.

Internara is a high-fidelity project, and we prioritize structured, technical communication.

### 1. Technical Documentation (Recommended)
Before seeking external help, please consult the authoritative documentation in the `docs/` directory:
- **[Software Requirements](docs/software-requirements.md)**: For functional understanding.
- **[System Architecture](docs/system-architecture.md)**: For technical structure and orchestration.
- **[Engineering Conventions](docs/engineering-standards.md)**: For development standards and patterns.

### 2. GitHub Issues (Bugs & Feature Requests)
If you encounter a bug or have a specific technical feature request:
- Search existing [Issues](https://github.com/reasnov/internara/issues) to see if it's already being addressed.
- Use our issue templates to provide a clear description and reproduction steps.
- Tag the issue with the relevant domain module (e.g., `Internship`, `Assessment`).

### 3. GitHub Discussions (General Help & Strategy)
For general questions, implementation advice, or architectural discussions:
- Start a thread in [Discussions](https://github.com/reasnov/internara/discussions).
- Engage with the community and maintainers to share ideas and strategy.
