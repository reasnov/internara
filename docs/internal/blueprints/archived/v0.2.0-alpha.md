# Application Blueprint: v0.2.0 (Core Engine)

**Series Code**: `ARC01-CORE-01` **Status**: `Archived` (Released)

> **Spec Alignment:** This blueprint lays the foundation for the **Platform and Technology**
> standards (Specs Section 5), including centralized database architecture and role-based access.

---

## 1. Core Problem Statement

Duplication of utility logic (UUIDs, Statuses). Lack of unified RBAC.

**Goal**: Establish the "Engine Room" (Shared & Core) for universal concerns.

---

## 2. Architectural Impact

### 2.1 Domain: Shared Utilities

- **New Module**: `Shared`.
- **Scope**: `HasUuid`, `HasStatuses`, `EloquentQuery`.
- **Identity:** Enforces UUIDs as the standard ID format.

### 2.2 Domain: Portable RBAC

- **New Module**: `Permission`.
- **Scope**: RBAC infrastructure supporting Instructor, Staff, Student roles.

---

## 3. UI/UX Strategy

- **i11n:** Permission labels support localization.

---

## 4. Contract Definitions

| Contract        | Owner Module | Consumer    | Purpose                                              |
| :-------------- | :----------- | :---------- | :--------------------------------------------------- |
| `EloquentQuery` | `Shared`     | All Domains | Standardizes CRUD operations and advanced filtering. |

---

## 5. Exit Criteria

- [x] **Universal UUIDs**: All models use `HasUuid`.
- [x] **Base Service**: Domain services inherit standard features.
- [x] **Spec Verification**: RBAC architecture supports required user roles.
