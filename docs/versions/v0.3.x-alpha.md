# Overview: Version v0.3.x-alpha (User & Profile Management)

## 1. Version Details

-   **Name**: `v0.3.x-alpha`
-   **Series Code**: `ARC01-USER`
-   **Status**: `In Progress`
-   **Description**: This version focuses on implementing the complete user lifecycle management and personalized profile systems. It builds upon the RBAC and Shared systems established in the previous phase to provide a secure and user-centric experience.

---

## 2. Goals & Architectural Philosophy

The objective of `v0.3.x-alpha` is to transition from system infrastructure to user-facing core features. We aim to provide a robust User Management system for administrators and a self-service Profile system for students, teachers, and staff.

### Problem Keypoints

-   **Fragmented User Data:** Ensuring all user types (Admin, Student, Teacher) share a consistent base model while having specialized profiles.
-   **Manual Account Provisioning:** The need for automated and secure user creation workflows.
-   **Auth Experience:** Refining the authentication and verification flows for a production-ready feel.

### Architectural Pillars

-   **Identity & Access Integration:** Leveraging the `Permission` module for all user-related authorization.
-   **Profile Polymorphism:** Handling specialized profile data through modular relations.
-   **Self-Service UI:** Providing intuitive Livewire components for profile editing and security settings.

## 4. Completed Milestones Details

### 4.1. UI Module Documentation (Audit)
-   Categorized all Blade components into **Layouts, Forms, Navigation, Display, and Utilities**.
-   Created a dedicated reference guide for each category in `docs/main/ui/components/`.

### 4.2. User & Profile Integration
-   **UserService Enhancement:** Integrated `ProfileService` to automatically initialize profile records for every new user.
-   **Role-Specific Data:** The administrative User Form now dynamically displays **NIP** (for Teachers) and **NISN** (for Students) based on selected roles.
-   **Status Management:** Integrated `spatie/laravel-model-status` for account activation/deactivation.
